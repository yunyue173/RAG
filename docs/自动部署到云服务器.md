# 自动部署到云服务器

本项目是 Spring Boot 动态网站，不能直接部署到 GitHub Pages。推荐方案是：

```text
GitHub 保存代码
GitHub Actions 自动打包
云服务器运行 jar
```

完成后，你以后只需要提交并推送代码：

```bash
git add .
git commit -m "update"
git push origin main
```

GitHub Actions 会自动构建并重启服务器上的网站。

## 一、准备云服务器

推荐使用 Ubuntu 22.04/24.04 服务器。需要在云平台安全组中放行：

```text
22    SSH
8080  网站访问端口
```

假设服务器信息如下：

```text
服务器公网 IP：你的服务器公网IP
服务器用户名：ubuntu
```

## 二、准备 SSH 部署密钥

在你自己的 Windows PowerShell 执行：

```powershell
ssh-keygen -t ed25519 -C "github-actions-rag-doc-qa" -f "$env:USERPROFILE\.ssh\rag_doc_qa_deploy"
```

把公钥复制到服务器：

```powershell
Get-Content "$env:USERPROFILE\.ssh\rag_doc_qa_deploy.pub"
```

登录服务器后，把这段公钥追加到：

```bash
mkdir -p ~/.ssh
nano ~/.ssh/authorized_keys
chmod 700 ~/.ssh
chmod 600 ~/.ssh/authorized_keys
```

测试能否免密登录：

```powershell
ssh -i "$env:USERPROFILE\.ssh\rag_doc_qa_deploy" ubuntu@你的服务器公网IP
```

## 三、初始化服务器

把初始化脚本上传到服务器：

```powershell
scp -i "$env:USERPROFILE\.ssh\rag_doc_qa_deploy" deploy/install-server.sh ubuntu@你的服务器公网IP:/tmp/install-server.sh
```

登录服务器执行：

```bash
chmod +x /tmp/install-server.sh
APP_USER=ubuntu /tmp/install-server.sh
```

然后编辑环境变量文件：

```bash
sudo nano /etc/rag-doc-qa/rag-doc-qa.env
```

填入你的 LLM 配置，例如 DeepSeek：

```text
APP_PORT=8080
RAG_STORAGE_DIR=/opt/rag-doc-qa/data/rag-store
LLM_API_KEY=你的DeepSeek_API_Key
LLM_BASE_URL=https://api.deepseek.com
LLM_CHAT_MODEL=deepseek-chat
LLM_EMBEDDING_ENABLED=false
```

注意：不要把真实 API Key 写进 GitHub 仓库。

## 四、推送项目到 GitHub

如果项目还不是 Git 仓库，在 `D:\system\system` 执行：

```powershell
git init
git add .
git commit -m "initial commit"
git branch -M main
git remote add origin https://github.com/你的GitHub用户名/你的仓库名.git
git push -u origin main
```

如果 GitHub 要求登录，按提示完成即可。

## 五、配置 GitHub Secrets

打开 GitHub 仓库：

```text
Settings -> Secrets and variables -> Actions -> New repository secret
```

添加：

```text
SERVER_HOST      你的服务器公网IP
SERVER_USER      ubuntu
SERVER_PORT      22
SERVER_SSH_KEY   私钥内容
```

`SERVER_SSH_KEY` 填私钥内容，不是公钥。Windows 查看私钥：

```powershell
Get-Content "$env:USERPROFILE\.ssh\rag_doc_qa_deploy"
```

可选：

```text
APP_DIR          /opt/rag-doc-qa
SERVICE_NAME     rag-doc-qa
```

不填可选项时，GitHub Actions 会使用默认值。

## 六、自动部署

之后每次推送到 `main` 或 `master` 分支，都会自动部署：

```powershell
git add .
git commit -m "修改说明"
git push origin main
```

查看部署过程：

```text
GitHub 仓库 -> Actions -> Deploy RAG Doc QA
```

部署成功后访问：

```text
http://你的服务器公网IP:8080
```

## 七、服务器常用命令

查看运行状态：

```bash
sudo systemctl status rag-doc-qa --no-pager
```

查看实时日志：

```bash
sudo journalctl -u rag-doc-qa -f
```

重启服务：

```bash
sudo systemctl restart rag-doc-qa
```

停止服务：

```bash
sudo systemctl stop rag-doc-qa
```

## 八、改代码后还要重新部署吗

要重新部署，但现在是自动的：

```text
你 push 代码 -> GitHub Actions 自动打包 -> 自动上传服务器 -> 自动重启网站
```

所以你不需要手动上传 jar。
