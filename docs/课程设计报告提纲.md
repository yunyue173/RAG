# 课程设计报告提纲

## 1. 课题信息

- 课题名称：基于 RAG 的智能文档问答系统
- 开发工具：IntelliJ IDEA、JDK 21、Maven
- 开发框架：Spring Boot、Spring WebMVC、Thymeleaf

## 2. 需求分析

### 2.1 项目背景

检索增强生成（RAG）通过“文档检索 + 大模型生成”的方式减少回答幻觉，适合企业知识库、课程资料问答、制度文档问答等场景。

### 2.2 功能需求

- 用户上传 PDF 或 TXT 文档。
- 系统解析文档文本并按固定大小切块。
- 系统对文本块生成向量并保存。
- 用户在聊天界面提问，系统检索 Top-K 相关片段。
- 系统基于检索结果生成答案，并展示引用原文。
- 同一文档支持多个会话，历史消息不跨文档。

## 3. 系统设计

### 3.1 总体架构

前端页面通过 REST API 调用后端。后端分为文档解析、文本切块、向量生成、向量检索、答案生成、历史存储等模块。

### 3.2 模块划分

- `DocumentParserService`：解析 PDF/TXT。
- `TextSplitterService`：文本分块与重叠切分。
- `EmbeddingService`：生成文本向量。
- `VectorSearchService`：余弦相似度 Top-K 检索。
- `AnswerGeneratorService`：LLM 回答或本地抽取式回答。
- `RagRepository`：文档、分块、会话、消息的本地持久化。
- `ApiController`：提供前后端交互接口。

### 3.3 数据设计

- `DocumentRecord`：文档编号、文件名、大小、字符数、分块数、上传时间。
- `DocumentChunk`：分块编号、所属文档、原文内容、起止位置、向量。
- `Conversation`：会话编号、所属文档、标题、创建时间、更新时间。
- `ChatMessage`：消息编号、所属会话、角色、内容、引用片段、创建时间。
- `Citation`：引用分块编号、原文内容、相似度。

## 4. 关键实现

### 4.1 文档上传与解析

前端使用 `XMLHttpRequest` 监听上传进度，后端通过 `MultipartFile` 接收文件。PDF 使用 Apache PDFBox 提取文本；TXT 优先 UTF-8 解码，失败后使用 GB18030 兼容中文文本。

### 4.2 文本分块

默认分块大小为 500 字符，重叠 80 字符。分块时优先在句号、问号、感叹号、换行位置断开，尽量保留语义完整性。

### 4.3 向量检索

每个文本块保存一个向量。提问时先将问题向量化，再计算问题向量与当前文档全部文本块向量的余弦相似度，返回 Top-K 片段。

### 4.4 答案生成与引用展示

如果配置 `OPENAI_API_KEY`，系统调用大模型基于上下文片段生成答案；如果未配置，则使用本地抽取式回答。前端展示答案、引用原文块和相似度，并高亮问题关键词。

## 5. 测试方案

- 上传 TXT 文件，验证解析、分块、向量保存是否成功。
- 上传 PDF 文件，验证 PDFBox 是否能提取正文。
- 针对文档内容提问，验证 Top-K 引用是否与问题相关。
- 新建多个会话，验证历史消息只在当前文档内生效。
- 删除文档，验证文档分块、会话、消息同步删除。

## 6. 总结与改进

当前系统已实现完整 RAG 闭环。后续可扩展为 Chroma、FAISS 或 Milvus Lite 等专用向量数据库；也可以增加用户登录、权限隔离、更多文件格式解析、流式输出和文档片段定位预览。
